# LLM Chat with SymPy MCP - 混合推理模式

## 核心概念

这个应用实现了 **LLM 与 SymPy 的混合推理模式**：

- **LLM** 负责：理解问题、组织语言、逻辑推理、决定何时需要计算
- **SymPy** 负责：提供精确的数学计算结果
- **用户** 看到：完整的、连贯的回复，工具调用过程完全透明

## 工作原理

### 典型对话流程（以"讲解乘法"为例）

```
用户：请给我讲解一下乘法，并举几个例子

┌─────────────────────────────────────────┐
│ 第 1 轮：LLM 分析                         │
│ LLM 决定："我需要举例说明，比如 4*2"     │
│ LLM 返回：调用工具 sympy_calculate("4*2")│
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ 系统执行 SymPy                           │
│ 计算：4*2 = 8                            │
│ 将结果返回给 LLM                         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ 第 2 轮：LLM 继续推理                     │
│ LLM 拿到结果 "8"，继续生成完整回复：     │
│ "乘法是... 例如 4*2=8，表示..."          │
└─────────────────────────────────────────┘
              ↓
         用户看到完整回复
```

### 关键技术点

1. **Tool Calling（工具调用）**
   - 使用 OpenAI 兼容的 Function Calling API
   - LLM 自主决定何时调用数学工具
   - 支持多次迭代调用

2. **混合推理循环**
   ```python
   while iteration < max_iterations:
       # LLM 分析并可能请求工具调用
       response = llm(messages, tools=enabled)
       
       if response.has_tool_calls:
           # 执行工具调用（SymPy 计算）
           results = execute_tools(response.tool_calls)
           # 将结果返回给 LLM
           messages.append(tool_results)
           # 继续下一轮，LLM 基于精确结果继续推理
       else:
           # LLM 生成最终回复
           return response.content
   ```

3. **对用户透明**
   - 用户只看到最终的完整回复
   - 工具调用过程在后台进行
   - 控制台会显示调试信息（可选）

## 支持的数学运算

SymPy 支持广泛的数学运算：

### 基础运算
- 加减乘除：`4*2`, `10/3`
- 幂运算：`2**10`, `3^2`
- 平方根：`sqrt(16)`
- 三角函数：`sin(pi/2)`, `cos(pi)`

### 高级运算
- 求导：`diff(x**2, x)` → `2*x`
- 积分：`integrate(x**2, x)` → `x**3/3`
- 解方程：`solve(x**2 - 4, x)` → `[-2, 2]`
- 化简：`simplify((x+1)**2 - x**2)` → `2*x + 1`

## 为什么需要混合推理？

### 问题：纯 LLM 的计算不可靠

```
用户：345 * 678 等于多少？

纯 LLM 可能回答：234,560 (错误❌)
正确答案：233,910
```

LLM 是基于概率生成文本，不是真正的计算。

### 解决方案：LLM + SymPy

```
用户：345 * 678 等于多少？

混合推理流程：
1. LLM 识别这是数学计算
2. LLM 调用工具：sympy_calculate("345*678")
3. SymPy 精确计算：233910
4. LLM 基于正确结果回答："345 * 678 = 233,910"
```

✅ 结果准确可靠！

## 扩展性

可以轻松添加更多工具：

- 代码执行工具
- 搜索引擎工具
- 数据库查询工具
- 其他科学计算库

只需在 `_build_tool_definition()` 中添加新的工具定义，
并在 `_execute_tool()` 中实现对应的执行逻辑。

## 注意事项

1. **API 兼容性**：需要支持 Function Calling 的 LLM API
   - OpenAI GPT-4/GPT-3.5
   - Azure OpenAI
   - 通义千问（部分模型）
   - 其他兼容 OpenAI 格式的 API

2. **最大迭代次数**：默认 5 次，防止无限循环
   - 可在 `LLMChatWithMath.max_tool_iterations` 调整

3. **调试信息**：控制台会显示工具调用过程
   - 生产环境可以关闭或记录到日志文件
